Commit and push to GitHub without running tests or checks.

1. Stage all changed files relevant to the current work.
2. Create a conventional commit message (feat/fix/refactor/docs/chore) with a concise summary of the changes.
3. Push to the current branch.
4. Report the commit hash and branch name.

Do NOT run typecheck, eslint, format:check, lint, or any other checks before committing.
